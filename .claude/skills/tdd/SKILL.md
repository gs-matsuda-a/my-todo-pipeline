# TDD スキル

## サイクル
```
RED → GREEN → REFACTOR → REPEAT
RED:      失敗するテストを先に書く
GREEN:    テストを通す最小限のコードを書く
REFACTOR: テストを保ったままコードを改善
```

## 手順
1. **ユーザージャーニー**: 「[ロール]として[アクション]したい（[理由]）」を1行で。
2. **インターフェース定義**: 公開する型・関数のシグネチャだけ書き、本体は未実装にする。
3. **RED**: プロジェクトのテストフレームワークで失敗するテストを書く。
4. **テスト実行（失敗を確認）**: 言語を自動判定（JS/TS:`npm test` / Python:`pytest` / Go:`go test ./...` / Rust:`cargo test`）。
5. **GREEN**: テストを通す最小限の実装。
6. **テスト実行（成功を確認）**。
7. **REFACTOR**: 品質改善。公開シンボルの docstring も整える。
8. **カバレッジ検証**: 80%以上を確認（認証・課金・セキュリティ重要コードは100%）。

## 必ず書くエッジケース
null/空値・空コレクション・境界値・不正な型・エラー（ネットワーク/DB）・大量データ・特殊文字。
