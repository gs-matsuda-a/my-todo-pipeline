# Plan スキル

実装に入る前に**計画と実装を分離**する。明示的な承認があるまでコードは書かない。

## 使い方
```
/plan
/plan ユーザー認証機能を追加
```

## 出力する内容
1. **要件の再確認** — 何を作るかを1〜数行で明確化
2. **リスク** — 想定エラー・ブロッカー・影響範囲
3. **段階計画（依存グラフ）** — 各ステップに以下を付与:

| フィールド | 説明 | 例 |
|-----------|------|-----|
| `step_id` | 一意ID | `S1` |
| `title` | タイトル | `CSV Parser 実装` |
| `depends_on` | 依存する step_id | `[S0]` |
| `files` | 変更ファイル | `[src/csv.ts, src/csv.test.ts]` |
| `type` | `setup`/`feature`/`integration`/`verification` | `feature` |

依存から**並列実行可能なレベル**を導出する:
```
Level 0: [S0]        ← setup
Level 1: [S1, S2]    ← 並列可
Level 2: [S3]        ← Level1完了後
```

4. **API 仕様（該当時）** — エンドポイントを追加/変更する場合、メソッド・パス・認証・req/res スキーマ・エラーコードを提示。
5. **確認待ち** — 「この方針で実装してよいか」をユーザーに確認する。

## 連携
- `/issue-flow` の Phase 1 から自動で呼ばれる
- 承認後は `/tdd` で実装
