# UI デザイン改善 + タスク編集機能
> `/issue-create docs/requirements/ui-redesign-task-edit.md` で開発に渡せます。

## 1. 目的・背景

**現状**: 既存の Todo アプリは基本機能（作成・完了・削除・共有）を持つが、UI が素朴でユーザー体験が物足りない状態。タスク作成後の修正には削除→再作成という手順が必要。

**課題**: 
- UI が古く見える（グラデーション基調だが、コンポーネントが平坦）
- タスク管理に修正機能がなく、CRUDとして不完全
- インタラクティブ性に欠ける（アニメーションなし）

**解決**: モダンで洗練された UI に刷新し、完全な CRUD 機能を実装することで、ユーザーの満足度と利便性を向上させる。

---

## 2. アクターとユースケース

| アクター | 役割 | 主な操作 |
|----------|------|--------|
| 一般ユーザー | タスクを作成・管理・編集・削除する | ログイン後、タスク一覧でタイトルや期限を編集 |
| 共有相手 | 共有されたタスクを閲覧・編集する | 共有相手が編集権限で受け取ったタスクを修正 |

### UC-1: タスクをインライン編集する
**アクター**: 一般ユーザー  
**トリガー**: タスク一覧でタイトルまたは期限を修正したい  
**基本フロー**:
1. ユーザーが一覧内のタスク行にマウスオーバーで編集UI が出現
2. 「編集」ボタンをクリック、またはタイトル/期限フィールドをダブルクリック
3. 該当フィールドがインライン入力欄に変わる
4. ユーザーが新しい値を入力し Enter / クリック外で確定
5. サーバに PUT リクエストを送信、確定後は表示に反映
6. アニメーション（フェード / スライド）で変更を視覚化

**代替フロー**:
- Cancel（Esc キー）で編集をキャンセル。表示は元に戻る。
- 入力エラー（タイトル空白・期限形式違い）は inline error message で表示。

### UC-2: タスク作成後すぐに期限を設定する
**アクター**: 一般ユーザー  
**トリガー**: 作成直後にタスク詳細を補完したい  
**基本フロー**:
1. ユーザーが「新しいタスクを入力...」で title を入力し「追加」をクリック
2. 新タスクがアニメーション（フェード イン）で一覧トップに表示
3. ユーザーが期限フィールドをクリックして日付入力欄をアクティベート
4. 日付を入力、Enter で確定
5. 確定後はタスク行に期限表示

## 3. 機能要件

### F-1: モダン UI デザイン
- **グラデーション + 丸いボタン**: 背景グラデーション（既存）+ ボタンは border-radius を大きく（15〜24px）、ソフトシャドウを追加
- **配色**: グラデーション基調は保持、コンポーネント間の視覚的メリハリを強調（背景色・テキスト色・ボタン色のコントラスト）
- **タイポグラフィ**: フォント族・ウェイト・サイズを整理し、階層を明確化

**受け入れ条件**:
- [ ] すべてのボタン（追加・削除・共有・編集確定）が border-radius で丸みを帯びている
- [ ] ボタンホバー時に色変更 + スケール（微細な拡大）が発生
- [ ] コンテナ・カード要素に box-shadow でデプスを表現

### F-2: アニメーション・トランジション
- **ホバー**: ボタン・リンク要素に `hover` で color/background/transform トランジション
- **追加・削除**: タスクリストへの追加時はフェード イン（opacity: 0 → 1、0.3s）、削除時はスライドアウト（slideOut）
- **編集開始・確定**: フィールド変更時はフェード イン・アウト（切り替わり時）、アニメーション時間 0.2〜0.3s
- **モーダル**: 共有モーダル出現時はフェード + スケール（scale-up）

**受け入れ条件**:
- [ ] ボタンホバーで transition（color/background 0.3s、transform 0.2s）が視認できる
- [ ] タスク追加時に新行がフェード イン（0.3s）で出現
- [ ] タスク削除時にスライドアウト（または フェード アウト）でアニメーション削除
- [ ] 編集モード切り替え時、フィールドがスムーズに遷移する

### F-3: タスク編集機能（インラインエディット）
- **編集対象**: `title` と `due_date` （`completed` はチェックボックスで既に可能なため対象外）
- **編集方式**: インラインエディット（タスク行の該当フィールドが input に変わる）
- **トリガー**: 
  - 編集ボタン（新規追加）でモード開始
  - ダブルクリック（フィールド直接起動）でも可
- **確定**: Enter キー / フォーカスロス（blur）/ 確定ボタンで POST / PUT に送信
- **キャンセル**: Esc キー / キャンセルボタンで編集終了、値は元に戻す

**受け入れ条件**:
- [ ] タスク行に編集ボタンがあり、クリックで title と due_date がインラインで編集可能になる
- [ ] Enter で確定時、サーバに PUT /api/tasks/:id リクエストが送信される
- [ ] due_date は date input type で日付ピッカーが使える（ブラウザネイティブ）
- [ ] 編集中に Esc で元の表示に戻る（サーバ送信なし）
- [ ] タイトル空白・期限形式違いは入力時に reject（400）、inline error を表示

### F-4: 権限制御
- **自分のタスク**: 自分だけが編集可
- **共有タスク（閲覧のみ）**: 編集ボタン非表示、read-only 表示
- **共有タスク（編集可）**: 共有相手も編集ボタン有効、PUT リクエストを受け付ける

**受け入れ条件**:
- [ ] 自分が作成したタスク、または編集権限で共有されたタスクは編集可能
- [ ] 閲覧のみで共有されたタスクは編集ボタン非表示
- [ ] サーバ側で PUT リクエスト時に権限チェック（403 Forbidden）

## 4. データモデル

**既存テーブル（tasks）**:
```
id (PK) | user_id | title | due_date | completed | created_at | updated_at | ...
```

**変更**: なし。既存スキーマで対応可能。

**ビジネスルール**:
- タスク編集時、`updated_at` を現在時刻に更新
- `due_date` は NULL 許容（日付未設定のタスク）
- 編集権限は share テーブルの `permission` カラム（`view` / `edit`）で判定

## 5. 非機能要件

| 観点 | 要件 |
|------|------|
| **ブラウザ対応** | 最新 2 世代（Chrome / Edge / Safari）。IE 11 は非対応 |
| **デバイス対応** | PC 中心。スマートフォン対応は後来 (V2 扱い) |
| **パフォーマンス** | インラインエディット開始 < 200ms、PUT 確定 < 500ms（ネットワーク含む） |
| **アクセシビリティ** | 現状維持（既存 a11y 対応は継承） |
| **セキュリティ** | 既存対策（入力検証・認証・認可）を保持。編集時も PUT エンドポイントで検証 |

---

## 6. スコープ外

| 項目 | 理由 | 対応時期 |
|------|------|--------|
| **スマートフォン完全対応** | デバイス市場の優先度が PC > SP 段階 | V2（後来） |
| **リアルタイム同期（WebSocket）** | 単一ユーザー・複数デバイス同期の需要が未確認 | 要望確認後 |
| **ダークモード** | モダンデザイン実装後の追加テーマとして後来 | V1.5 以降 |

## 7. 用語集

| 用語 | 定義 |
|------|------|
| **インラインエディット** | ページ遷移なく、行内でフィールドを入力欄に変更して編集する UX |
| **編集権限** | 共有タスク受け取り時の権限レベル。`view` = 閲覧のみ、`edit` = 修正可 |
| **due_date** | タスクの期限（YYYY-MM-DD 形式）。NULL 許容 |
| **モダン UI** | グラデーション・丸いコンポーネント・アニメーションを備えた現代的な見た目 |
